# 快速开始：Telegram 影视资源采集

## 目标

验证 Telegram 影视资源采集功能的最小闭环，包括来源登记、增量采集、资源去重、链接追加和结果查询。

## 前置条件

1. 已配置数据库连接并可执行 Alembic 迁移。
2. 已准备可读取目标 Telegram 群聊的授权凭据。
3. 应用已安装本项目依赖并可运行测试。

## 验证步骤

1. 执行数据库迁移，确认采集来源、原始消息、影视资源和网盘链接相关表已创建。
2. 通过来源管理接口创建一个启用状态的 Telegram 群聊来源，并设置采集周期。
3. 手动触发一次该来源的增量采集。
4. 核查数据库或查询接口，确认新消息被保存为原始消息记录。
5. 核查资源查询接口，确认至少一条影视资源主记录已生成，并包含名称、类型、封面或待整理状态。
6. 在同一来源或其他来源准备一条指向同一影视资源但包含新网盘链接的消息，再触发一次采集。
7. 确认系统未创建重复主资源，而是在原资源下追加新的网盘链接。
8. 再次触发包含完全重复消息的采集，确认不会新增重复原始消息、重复资源或重复链接。

## 测试重点

- 同一消息重复采集是否幂等。
- 同一影视资源跨消息、跨群聊出现时是否正确归并。
- 不完整消息是否仍保留待整理记录。
- 资源详情是否可追溯到来源消息和来源群聊。

## 实现完成后的验证命令

```powershell
pytest
```

## 交付检查

- Alembic 迁移存在且可执行。
- 新增测试覆盖领域规则、应用用例和接口契约。
- `pytest` 通过后方可宣称实现完成。
